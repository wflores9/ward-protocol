9-Step Claim Validation

All checks query live XRPL state. Any failure returns CLAIM_REJECTED (422) with reason code and failed check index. No oracle. No human override.

01Policy NFT exists, owned by claimantNFT_NOT_FOUND
02Policy unexpired — XRPL ledger time, not server clockPOLICY_EXPIRED
03Vault address matches NFT URIVAULT_MISMATCH
04Default confirmed × 3 consecutive ledger closesDEFAULT_NOT_CONFIRMED
05Collateral ratio breached (health_ratio < 1.5)COLLATERAL_SUFFICIENT
06No active escrow pending for this policyESCROW_ALREADY_PENDING
07KYC credential (XLS-70) valid and unexpiredKYC_CREDENTIAL_INVALID
08Domain credential (XLS-80) valid for this institutionDOMAIN_NOT_AUTHORIZED
09No duplicate claim in rate-limit windowRATE_LIMIT_EXCEEDED
Checks 4–5 require XLS-66 on mainnet. Validated with mock fixtures on Altnet (106/106 passing).
Security

Security Invariants

15 attack vectors in security_notes.md. Core invariant: ward_signed = False.

No custody / no keys
Institutions sign with their own wallets. No wallet in Ward codebase.
Non-transferable policies
Policy NFTs minted without tfTransferable. XRPL enforces — not Ward policy.
PREIMAGE-SHA-256 escrow
Only the claimant holding the preimage can finish the escrow. Ward never sees it.
Ledger-time expiry
All expiry logic uses XRPL ledger time. Clock manipulation impossible.
Multi-ledger confirmation
Default events verified across 3 ledger closes. No single-block trigger.
Replay protection
Policy NFT burns atomically with EscrowFinish. Burned policy fails Check 01.
