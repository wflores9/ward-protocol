# Ward Protocol v0.3 — Roadmap

## Pathfinder Integration

- Update ward/primitives.py — UnsignedTransaction to support Paths and SendMax fields
- Add ripple_path_find pre-check in resolution logic when collateral asset != payout asset
- Update Step 9 solvency check to account for path availability
- Add partial resolution flag when no liquid path exists at ledger close
- Add tests covering multi-asset resolution scenarios

## Goal
Ward resolution transactions become atomic multi-asset settlements via XRPL native pathfinding. No external swap. No oracle. One transaction, one ledger close.

## Status
Planned. Not in v0.2.5 scope.

## XRPL EVM Sidechain — Solidity Port

- Deployment guide (`docs/integration/xrpl-evm.md`) — chain IDs, Hardhat config, oracle flow
- RLUSD-native on XRPL EVM Sidechain (chain ID 1440002 testnet / 1440001 mainnet)
